1. סקירה כללית

AlonBot הוא עוזר AI אישי ועסקי שרץ דרך Telegram. הוא משלב את Claude API של Anthropic עם מגוון כלים מעשיים — חיפוש באינטרנט, ניהול משימות, שליטה במחשב, יצירת תמונות, שליחת מיילים, ועוד.

יכולות עיקריות

  • שיחת AI חכמה — Claude Sonnet 4 עם fallback אוטומטי ל-Gemini
  • הודעות קוליות — תמלול אוטומטי (Whisper) + תשובה קולית (ElevenLabs)
  • זיכרון מתקדם — זוכר עובדות, העדפות, אירועים עם חיפוש סמנטי
  • יצירת תמונות — Gemini Flash ישירות לצ'אט
  • ניתוח תמונות ומסמכים — Vision API לתמונות, OCR, PDF
  • חיפוש אינטרנט — DuckDuckGo + מחקר מעמיק עם Gemini
  • Monday.com — שאילתות GraphQL ללוחות עסקיים
  • ניהול משימות — רשימת TODO עם עדיפויות ותאריכי יעד
  • תזכורות Cron — תזמון הודעות עתידיות
  • שליחת מיילים — דרך Gmail לכתובות מורשות
  • שליטה במחשב — פקודות shell, קריאת/כתיבת קבצים, צילום מסך
  • מעקב עלויות — דוח עלות API בזמן אמת
  • התראות פרואקטיביות — משימות שעבר זמנן, סיכום שבועי
  • מצב שקט — תשובות קצרות בלילה ובשבת

3. התקנה מהירה

דרישות מקדימות

  • Node.js 20+ (node -v)
  • npm (npm -v)
  • Git (git --version)
  • חשבון Anthropic עם API key
  • בוט Telegram (מ-@BotFather)

שלבי התקנה

1

שכפול הפרויקט

git clone https://github.com/alonr-create/alonbot.git
cd alonbot
2

הרצת סקריפט התקנה

./scripts/setup.sh

הסקריפט בודק Node.js, מתקין dependencies, מעתיק .env.example ל-.env, ובונה TypeScript.

3

הגדרת API Keys

# ערוך את .env עם המפתחות שלך
nano .env

מינימום נדרש: ANTHROPIC_API_KEY, TELEGRAM_BOT_TOKEN, ALLOWED_TELEGRAM

4

הרצת הבוט

# פיתוח (auto-reload):
npm run dev

# פרודקשן:
npm run build && npm start
5

בדיקה

שלח הודעה לבוט ב-Telegram. אם הכל עובד, תקבל תשובה עם footer שמציג את המודל והעלות.

חשוב!

לעולם אל תעלה את קובץ .env ל-Git. הוא מכיל מפתחות API רגישים. הקובץ כבר ב-.gitignore.

6. מערכת זיכרון

AlonBot משתמש במערכת זיכרון רב-שכבתית:

שכבות זיכרון

שכבהמהאיפהמשך
עבודה 20 הודעות אחרונות messages סשן
אפיזודי סיכומי שיחות conversation_summaries חודשים
סמנטי עובדות, העדפות, אירועים memories + vectors לתמיד

סוגי זיכרון

  • fact — עובדה ("יום הולדת: 15 למאי")
  • preference — העדפה ("אוהב סושי")
  • event — אירוע ("פגישה עם רו"ח מחר")
  • pattern — דפוס ("בודק מיילים בבוקר")
  • relationship — אדם ("דני — שותף עסקי")

קטגוריות

personal, work_dekel, work_mazpen, work_alon_dev, work_aliza, health, finance

חשיבות (1-10)

  • 1-3: מידע שולי (צבע אהוב, שיר ששמע)
  • 4-6: מידע רגיל (העדפות אוכל, הרגלים)
  • 7-8: מידע חשוב (פגישות, לקוחות)
  • 9-10: מידע קריטי (ימי הולדת, מידע עסקי מהותי)

Smart Retrieval

במקום לטעון את כל הזיכרונות, המערכת טוענת:

  1. כל הזיכרונות עם חשיבות 8+ (תמיד)
  2. זיכרונות שניגשו אליהם לאחרונה (< 7 ימים)
  3. חיפוש vector סמנטי על ההודעה (top 5)
  4. מקסימום 20 זיכרונות ב-prompt

סיכום אוטומטי

כל 40+ הודעות → Claude מסכם את השיחה ל-3-5 משפטים → נשמר ב-conversation_summaries עם נושאים.

תחזוקת זיכרון (03:00 יומי)

  • Decay: זיכרונות שלא ניגשו 60+ ימים → חשיבות -1
  • Consolidation: זיכרונות דומים מאוחדים
  • Cleanup: חשיבות 0 → מחיקה

7. Cron ותזמון

Cron מובנה

זמןתיאורסוג
03:00 יומי תחזוקת זיכרון (decay, consolidation) מערכת
08:00 יומי סיכום בוקר (תאריך, מזג אוויר, לידים, תזכורות) הודעה ל-Claude
18:00 יומי התראה על משימות שעבר זמנן ישירות
ראשון 09:00 סיכום שבועי (עלויות, משימות, לידים) הודעה ל-Claude

תזכורות מותאמות אישית

המשתמש יכול ליצור תזכורות cron דרך הצ'אט:

"תזכיר לי כל יום ב-9 בבוקר לבדוק מיילים"
→ set_reminder(name="בדיקת מיילים", cron_expr="0 9 * * *", message="תבדוק מיילים!")

תזכורות נשמרות ב-SQLite ושורדות restart.

תחביר Cron

┌──── דקה (0-59)
│ ┌──── שעה (0-23)
│ │ ┌──── יום בחודש (1-31)
│ │ │ ┌──── חודש (1-12)
│ │ │ │ ┌──── יום בשבוע (0-7, 0=ראשון)
│ │ │ │ │
* * * * *

דוגמאות:
0 9 * * *     = כל יום ב-09:00
0 9 * * 1-5   = ימי חול ב-09:00
*/30 * * * *  = כל 30 דקות
0 8 1 * *     = 1 בכל חודש ב-08:00

כל הזמנים באזור זמן ישראל (Asia/Jerusalem).

8. Cloud + Local Mac

AlonBot תומך בשני מצבים — ענן ומקומי — שעובדים יחד:

מצב ענן (MODE=cloud)
  • Telegram bot פעיל (polling)
  • Cron jobs רצים
  • כלים מקומיים עוברים proxy ל-Mac
  • רץ על Render/Docker/VPS
מצב מקומי (MODE=local)
  • Telegram בשליחה בלבד (ללא polling — מונע conflict)
  • Cron jobs רצים
  • כלים מקומיים רצים ישירות
  • חושף tool API דרך Express

הגדרת Cloudflare Tunnel

1

התקנת cloudflared

brew install cloudflared
2

הרצת הבוט מקומית

MODE=local npm run dev
3

הפעלת tunnel

./scripts/start-tunnel.sh

הסקריפט יוצר tunnel, מקבל URL, ורושם אותו אוטומטית בענן.

הפעלה אוטומטית (macOS)

# העתקת LaunchAgent
cp scripts/com.alonbot-tunnel.plist ~/Library/LaunchAgents/

# עריכת הנתיבים ב-plist
nano ~/Library/LaunchAgents/com.alonbot-tunnel.plist

# הפעלה
launchctl load ~/Library/LaunchAgents/com.alonbot-tunnel.plist

איך ה-Proxy עובד

  1. Mac עולה → tunnel נפתח → נרשם בענן (/api/register-local)
  2. משתמש מבקש צילום מסך → ענן מזהה כלי מקומי
  3. ענן שולח POST ל-Mac דרך tunnel (/api/tool)
  4. Mac מריץ → מחזיר תוצאה (כולל media ב-base64)
  5. ענן שולח תשובה ל-Telegram

9. Docker ו-Render

Docker

# בנייה
docker build -t alonbot .

# הרצה
docker run -d --name alonbot \
  --env-file .env \
  -e MODE=cloud \
  -v alonbot-data:/app/data \
  -p 3700:3700 \
  alonbot

ה-volume alonbot-data שומר את SQLite DB בין restarts.

Render

הפרויקט כולל render.yaml — Blueprint שמגדיר הכל:

  1. Push ל-GitHub
  2. ב-Render: New → Blueprint → חבר repo
  3. הגדר env vars בדשבורד
  4. Deploy (ידני או אוטומטי)
Render Free Tier

בתוכנית חינמית, השרת "נרדם" אחרי 15 דקות ללא תעבורה. הודעת Telegram הראשונה אחרי שינה תיקח 30-60 שניות. שקול תוכנית בתשלום ($7/חודש) לזמינות מלאה.

Health Check

GET /health
→ { "status": "ok", "mode": "cloud", "uptime": 3600, "localConnected": true }

10. אבטחה

AlonBot מיישם מספר שכבות אבטחה:

אימות משתמשים

  • רק מזהי Telegram/WhatsApp ב-ALLOWED_* יכולים לתקשר
  • הודעות ממשתמשים לא מורשים → "Unauthorized"
  • Rate limiting: 10 הודעות לדקה למשתמש

Shell Whitelist

רק פקודות בטוחות מורשות:

date, cal, ls, cat, head, tail, wc, grep, find, echo,
pwd, whoami, uptime, df, which, hostname, sw_vers

תווים מסוכנים חסומים: ; | & ` $ ( ) { } ! < >

הגבלת נתיבים

  • קריאה/כתיבה רק תחת תיקיות פרויקט
  • חסימת: .env, .git/, .ssh/, .claude/, credentials

מניעת SSRF

  • רק http: ו-https: מותרים
  • חסימת localhost, 127.0.0.1, 10.x, 192.168.x, 169.254.x

Email Whitelist

  • רק כתובות/דומיינים מורשים
  • ברירת מחדל: dprisha.co.il, gmail.com

Cloud-Local Bridge

  • אימות Bearer token ב-LOCAL_API_SECRET
  • Timeout 20 שניות לכל proxy call

Prompt Injection Protection

  • System prompt מנחה לעולם לא לעקוב אחרי הוראות מתוצאות כלים
  • הגבלת אורך הודעה: 4000 תווים
  • חסימת חשיפת מפתחות/סיסמאות

12. פתרון בעיות

בעיות נפוצות

הבוט לא מגיב
  • בדוק ש-TELEGRAM_BOT_TOKEN נכון
  • בדוק ש-ALLOWED_TELEGRAM מכיל את ה-ID שלך
  • בדוק ש-MODE=cloud (מצב local לא מקשיב להודעות)
  • בדוק לוגים: npm run dev
שגיאת 429 (Rate Limit)
  • Claude API מגביל ל-10K input tokens/דקה
  • הבוט עובר אוטומטית ל-Gemini fallback
  • אם גם Gemini כושל → המתן דקה ונסה שוב
"Mac לא מחובר"
  • בדוק שהבוט רץ מקומית: curl http://localhost:3700/health
  • בדוק שהtunnel פעיל: ./scripts/start-tunnel.sh
  • בדוק את ה-URL: cat /tmp/alonbot-tunnel-url.txt
better-sqlite3 לא מתקין
  • צריך build tools: xcode-select --install (Mac)
  • Docker: הDockerfile כבר כולל python3 make g++
הודעות קוליות לא עובדות
  • בדוק ש-GROQ_API_KEY מוגדר (לתמלול)
  • בדוק ש-ELEVENLABS_API_KEY מוגדר (ל-TTS)
  • ללא Groq: הבוט מבקש מהמשתמש לכתוב בטקסט
