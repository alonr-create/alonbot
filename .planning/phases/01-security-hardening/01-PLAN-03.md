# Plan 3: Cookie-Based Dashboard Auth & Separate Secrets

## Objective
Replace query-parameter token authentication with HttpOnly cookie-based sessions for the dashboard and chat pages, add rate limiting on auth failures, and split the shared `localApiSecret` into two separate secrets: one for the cloud-local bridge and one for the dashboard.

## Context
- Currently `dashAuth` middleware at `src/gateway/server.ts:70-77` checks `req.query.token` or `req.headers['x-dashboard-token']` against `config.localApiSecret`.
- The token appears in browser URL bar at `/dashboard?token=SECRET` and `/chat?token=SECRET` (lines 196-207, 212-213).
- Inline JS in dashboard HTML (line 303-304) stores the token as `const TOKEN = '...'` and passes it in every fetch URL.
- Chat HTML (lines 441, 450, 484, 510) does the same.
- The same `localApiSecret` is used for both dashboard auth and the cloud-local bridge (lines 26, 46, 72).
- Research provides a complete cookie auth implementation with rate limiting (`01-RESEARCH.md` section 3).
- User decisions: HttpOnly cookie, 7-day TTL, rate limit 5 failures per minute, redirect to strip token from URL.
- No new dependencies needed — manual cookie parsing is sufficient.

## Task 3: Replace dashAuth middleware

Replace the existing `dashAuth` function (lines 70-77) with the new cookie-based version. The flow:

1. **Check rate limit** — if IP is rate-limited, return 429.
2. **Check session cookie** — parse `alonbot_session` from cookies. If valid and not expired, call `next()`.
3. **Check query token** — if `req.query.token === config.dashboardSecret`:
   - Generate a new session token (`crypto.randomBytes(32).toString('hex')`)
   - Store in `sessions` map
   - Set `HttpOnly`, `Secure` (when behind HTTPS), `SameSite=Lax` cookie with 7-day `maxAge`
   - For HTML page requests (`/dashboard`, `/chat`): **redirect** to the same path without the token query param to strip it from the URL bar
   - For API requests: call `next()` directly
4. **Check header token** — if `req.headers['x-dashboard-token'] === config.dashboardSecret`, call `next()` (for programmatic API access).
5. **Auth failed** — record failure, return 401.

Important: use `config.dashboardSecret` (not `config.localApiSecret`) for dashboard auth.
