# Plan 2: Git Token Removal & Output Redaction

## Objective
Remove GITHUB_TOKEN from all git remote URLs so it never appears in `git remote -v` output or error messages, and add output redaction as defense-in-depth to strip any token patterns from tool output before it reaches Telegram.

## Context
- GITHUB_TOKEN is embedded in git push URLs at 6 locations in `src/agent/tools.ts`:
  - Line 857: `deploy_app` (vercel) — `https://${token}@github.com/...`
  - Line 877: `deploy_app` (railway) — same pattern
  - Line 940: `auto_improve` (cloud git push) — `https://${token}@github.com/...`
  - Line 975: `build_website` — `https://${token}@github.com/...`
  - Line 990: `build_website` execSync — pushUrl with embedded token
- GitHub API calls (lines 844-854, 865-875, 978-988) use `Authorization: Bearer ${token}` header — these are fine (not in output).
- Error messages at lines 887, 945, 996 include `e.stderr` which may contain the token if git push fails.
- Research recommends `GIT_ASKPASS` approach: a tiny shell script that echoes the token when git asks for credentials. This avoids embedding the token in URLs entirely.
- Output redaction catches any token patterns (ghp_, gho_, github_pat_, sk-ant-, Bearer tokens) as a safety net.

## Task 1: Create git-auth utility

Create `src/utils/git-auth.ts` with three exports as specified in `01-RESEARCH.md` section 2:

**`setupGitAuth()`** — writes a `GIT_ASKPASS` script to `/tmp/alonbot-git-askpass.sh` that echoes the GITHUB_TOKEN. Called once at startup. The script content:
```bash
#!/bin/sh
echo "${token}"
```
Use `writeFileSync` and `chmodSync('700')`.

**`gitEnv()`** — returns a copy of `process.env` with `GIT_ASKPASS` set to the script path and `GIT_TERMINAL_PROMPT=0`. This is passed as the `env` option to `execSync` for all git operations.

**`redactSecrets(output: string): string`** — replaces token patterns in output strings:
- GitHub tokens: `ghp_[a-zA-Z0-9]{36,}`, `gho_[a-zA-Z0-9]{36,}`, `github_pat_[a-zA-Z0-9_]{22,}`
- Embedded credentials in URLs: `://[^@]+@github.com` → `://[REDACTED]@github.com`
- Anthropic keys: `sk-ant-[a-zA-Z0-9-]{20,}`
- Bearer tokens in output: `Bearer [a-zA-Z0-9._-]{20,}`
- The actual GITHUB_TOKEN value itself (if set): `process.env.GITHUB_TOKEN` as a literal string match

## Task 6: Redact shell tool output

In the `case 'shell':` handler (line 278-284), wrap both success and error returns with `redactSecrets`:

```typescript
case 'shell': {
  // ... blocklist check from Plan 1 ...
  try {
    const output = execSync(input.command, { shell: '/bin/zsh', timeout: 30000, encoding: 'utf-8', maxBuffer: 1_000_000 }).trim();
    return redactSecrets(output);
  } catch (e: any) {
    return `Error: ${redactSecrets((e.stderr || e.message || '').slice(0, 1000))}`;
  }
}
```

This ensures that even if a user runs `git remote -v` or `env` via the shell tool, no tokens appear in the output sent to Telegram.
