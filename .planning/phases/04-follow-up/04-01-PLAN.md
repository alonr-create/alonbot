1. Add follow_ups table to `src/db/schema.ts` using the same idempotent CREATE TABLE IF NOT EXISTS pattern:
```sql
CREATE TABLE IF NOT EXISTS follow_ups (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  phone TEXT NOT NULL,
  message_number INTEGER NOT NULL CHECK(message_number IN (1, 2, 3)),
  scheduled_at TEXT NOT NULL,
  sent_at TEXT,
  cancelled INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_follow_ups_due
  ON follow_ups(scheduled_at) WHERE sent_at IS NULL AND cancelled = 0;
CREATE INDEX IF NOT EXISTS idx_follow_ups_phone
  ON follow_ups(phone) WHERE sent_at IS NULL AND cancelled = 0;
```

- `markFollowUpSent(id: number): void` — UPDATE sent_at = datetime('now')
- Export `FollowUpRow` type: `{ id, phone, message_number, scheduled_at, name, interest, status }`

    - If outside business hours, scheduler defers follow-up to next business day at 09:30
    - After sending follow-up #1, scheduler schedules #2 (day 3 total). After #2, schedules #3 (day 7 total)
    - After sending follow-up #3, no further follow-ups are scheduled

1. Create `src/follow-up/follow-up-ai.ts`:
- `generateFollowUpMessage(messageNumber: 1|2|3, leadName: string, interest: string): Promise<string>`
- Build a dedicated system prompt per message number:
  - #1: "friendly check-in, casual and warm" tone
  - #2: "value reminder with gentle urgency, mention other interested clients" tone
  - #3: "final message, respectful closing, leave door open" tone
- Call `generateResponse` with a single user message asking Claude to write the follow-up in Hebrew
- Keep messages short: 2-3 sentences max
- Do NOT mention this is automated

2. Create `src/follow-up/scheduler.ts`:
- `startFollowUpScheduler(sock: WASocket): void` — starts a 15-minute setInterval
- On each tick: get due follow-ups via `getDueFollowUps()`
- For each due follow-up:
  a. Check `isBusinessHours()` — if false, compute next business day via `getNextBusinessDay()`, add 30 minutes (09:30), update scheduled_at, continue
  b. Re-read the follow-up from DB to confirm not cancelled (race condition guard)
  c. Generate message via `generateFollowUpMessage(fu.message_number, fu.name, fu.interest)`
  d. Send via `sendWithTyping(sock, phone + '@s.whatsapp.net', message)`
  e. `markFollowUpSent(fu.id)`
  f. Store outgoing message in messages table (direction='out')
  g. If message_number < 3, schedule next follow-up: #1 -> #2 in 2 days, #2 -> #3 in 4 days (cascading)
- Run check() immediately on startup, then every 15 minutes
- Wrap entire check in try/catch, log errors via pino

3. Create tests in `src/follow-up/__tests__/scheduler.test.ts`:
- Mock `generateFollowUpMessage` to return a fixed string
- Mock `sendWithTyping` to capture calls
- Mock `isBusinessHours` to test both paths
- Test: due follow-up is sent and marked
- Test: outside business hours defers to next business day
- Test: after sending #1, #2 is scheduled; after #2, #3 is scheduled; after #3, nothing scheduled
- Test: cancelled follow-up is skipped
